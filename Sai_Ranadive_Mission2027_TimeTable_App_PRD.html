1. Purpose & Scope

Build a single, personal, password-protected web application — "Mission 2027" — that merges three parallel tracks into one interactive daily/weekly planner and tracker:

  1. UPSC CSE 2027 preparation (Prelims + Mains + Optional), driven by the Margdarshan IAS Micro-Planner content supplied.
  2. B.Tech CSE Semester-V academics at Symbiosis Institute of Technology — four exam subjects (Theory of Computation, Computer Networks, Design & Analysis of Algorithms, Financial Mathematics), each with 5 units (theory + practical), plus the fixed college timetable.
  3. Content creation — one weekly long-form YouTube video + daily Reels/Shorts on Maharashtra politics (Vidhan Sabha/Vidhan Parishad coverage), largely in Marathi.

The app must auto-generate Sai's daily schedule from his college timetable + fixed travel time, track completion of every planned block, and auto-reschedule anything missed instead of just showing it as failed. It will be built as a React + TypeScript + Tailwind single-page app, hosted free on Netlify, backed by a minimal Node/Express API hosted free on Render, with Neon Postgres as the database.

Correction to spec assumption: a browser cannot speak the Postgres wire protocol directly, so a thin API layer is mandatory to talk to Neon — it cannot be skipped. The good news: that layer is tiny (a handful of CRUD routes, see Section 13), can run on Render's free tier, and requires no heavy backend engineering. Node + Express (or Fastify) is the right minimal choice for it — this is not a contradiction of "no big backend," it is exactly the "no much backend needed" version.

8. Auto-Adjust / Rescheduling Engine

This is the core differentiator of the app: nothing that gets missed simply disappears or sits marked "failed" forever — it flows forward.

8.1 Rule 1 — Missed Task Rollover

IF a study_session's status is still "pending" after 23:59 on its planned_date:
    SET status = "missed"
    FIND next available slot of the SAME category (upsc_gs / academic / content)
        within the next 2 calendar days
    RE-INSERT the topic at that slot, shifting any already-queued topics
        for that category forward by one slot each (FIFO queue per subject)
    LOG the shift in an audit trail (original_date, new_date, reason)

8.2 Rule 2 — Scheduled Template Override (date-ranged)

overrides = [
  { from: "2026-10-15", to: "2026-11-26", template: "exam_sprint" }
]
ON every dashboard load / daily cron:
    activeTemplate = overrides.find(o => today BETWEEN o.from AND o.to)?.template
                     ?? "default"
    RENDER today's schedule using activeTemplate's block map (Section 6 table)

8.3 Rule 3 — Buffer & Pace Tracking

bufferDays = count(Sundays/holidays where all planned blocks were completed early)
backlogDays = count(pending micro_topics whose planned_date < today)
paceStatus =
    backlogDays == 0                      -> "green"  (on pace)
    backlogDays <= bufferDays              -> "amber"  (absorbable via banked buffer)
    backlogDays > bufferDays               -> "red"    (needs a catch-up weekend / plan reshuffle)
DISPLAY paceStatus as a coloured badge on the Dashboard next to
    "Days to Prelims" and "Days to Mains" counters.

8.4 Rule 4 — Streaks

Independent streak counters per category (UPSC / Academics / Content). A day counts toward the streak if ≥ 80% of that category's planned minutes for the day were logged as complete. Missing a day resets that category's streak to 0 but does not touch the other two.

12. Database Schema (Neon Postgres)

users(
  id UUID PK, name TEXT, dob DATE, upsc_attempt_year INT,
  prelims_date DATE, mains_date DATE, created_at TIMESTAMPTZ
)

settings(
  user_id UUID FK, wake_time TIME, sleep_time TIME,
  travel_minutes_each_way INT DEFAULT 60,
  optional_subject_name TEXT, notif_email BOOLEAN,
  updated_at TIMESTAMPTZ
)

subjects(
  id UUID PK, type TEXT CHECK (type IN ('academic','upsc_gs','upsc_optional','content')),
  name TEXT, exam_date DATE NULL, total_units INT NULL
)

micro_topics(
  id UUID PK, subject_id UUID FK, day_number INT, title TEXT,
  subtopics JSONB, resources JSONB,
  planned_date DATE, status TEXT CHECK (status IN ('pending','done','missed','rescheduled')),
  completed_at TIMESTAMPTZ NULL
)

timetable_slots(
  id UUID PK, day_of_week TEXT, start_time TIME, end_time TIME,
  slot_type TEXT CHECK (slot_type IN ('class','free','lunch')),
  label TEXT
)

daily_template_blocks(
  id UUID PK, template_name TEXT CHECK (template_name IN ('default','exam_sprint')),
  day_type TEXT CHECK (day_type IN ('weekday','saturday','sunday')),
  start_time TIME, end_time TIME, category TEXT, label TEXT, sort_order INT
)

study_sessions(
  id UUID PK, user_id UUID FK, date DATE, block_label TEXT,
  subject_id UUID FK NULL, planned_minutes INT, actual_minutes INT DEFAULT 0,
  status TEXT CHECK (status IN ('pending','done','missed','rescheduled')), notes TEXT
)

content_calendar(
  id UUID PK, date DATE, content_type TEXT CHECK (content_type IN ('reel','short','long_video')),
  title TEXT, stage TEXT CHECK (stage IN ('idea','scripted','shot','edited','published')),
  platform TEXT, publish_at TIMESTAMPTZ NULL
)

tasks(
  id UUID PK, date DATE, title TEXT, category TEXT,
  status TEXT CHECK (status IN ('pending','done')), priority TEXT
)

streaks(
  id UUID PK, category TEXT, current_streak INT, longest_streak INT, last_completed_date DATE
)

audit_log(
  id UUID PK, entity_id UUID, action TEXT, original_date DATE, new_date DATE, reason TEXT, created_at TIMESTAMPTZ
)

15. Non-Functional Requirements

  • Entire stack must run on free tiers (Netlify free, Render free web service, Neon free tier).
  • Mobile-responsive — Sai will check/tick tasks on his phone between college periods.
  • All writes optimistic in the UI (tick a box → instant visual feedback → background sync), so it feels realtime even on Render's free-tier cold starts.
  • No external user data collected — single-user personal tool; still, JWT + HTTPS only.
  • Dates/times must be handled in Asia/Kolkata timezone throughout, both frontend display and backend cron.
